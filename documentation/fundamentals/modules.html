Modules are self-contained packages that extend the core functionality of your application. They provide additional features, integrations, or behavioral enhancements without modifying the base system. They are included in Front and promote modularity, scalability, and a clean separation of concerns by allowing developers to enable or disable them as needed.

Activating Modules

Modules are loaded via the module attribute of the <script> element. You can activate one or multiple modules in your application.

Example:
FTML (index.html) <!doctype html> <html> <head> <!-- Load Front with navigate and animate modules --> <script src="https://cdn.front.nu/{ma}.{mi}.{pa}/front.js" module="navigate;animate"> </script> </head> <body> <main> <a href="page1.html">Go to Page 1</a> </main> </body> </html>
Note: Separate multiple modules using semicolons (;). Avoid using commas or spaces.

Configuring Modules

Modules that support configuration use the {module}-conf attribute on the <script> element. This sets default options for the module across the application.

Example:
FTML (index.html) <script src="https://cdn.front.nu/{ma}.{mi}.{pa}/front.js" module="navigate" navigate-conf="target:main;preloader:#navloader;startpage:my-app"> </script>
Note: Not all modules require configuration. Some modules like animate are configured entirely through per-element attributes.

Using Module Attributes

Modules expose attributes following the pattern {module}-{action}. These are placed on individual elements to trigger module behavior.

Example:
FTML <!-- Animate module: configure per element --> <div id="box" animate-start animate-key="0;50;100" animate-transform="scale(1);scale(1.5);scale(1)" animate-duration="2s" animate-easing="ease-in-out" animate-iteration="infinite"> Animated box </div>
