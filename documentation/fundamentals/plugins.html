Plugins are third-party components that extend the capabilities of your application by adding new features, integrations, or custom behavior. They can modify or enhance modules without changing the core system, allowing developers to easily add functionality and maintain a modular architecture.

Loading Plugins

Plugins are loaded via the plugin attribute of the <script> element. You can activate one or multiple plugins in your application.

Example:
FTML (index.html) <!doctype html> <html> <head> <!-- Load Front with syntaxhighlighting and scrollbar plugins --> <script src="https://cdn.front.nu/{ma}.{mi}.{pa}/front.js" plugin="syntaxhighlighting;scrollbar"> </script> </head> <body> <main> <code syntaxhighlighting--set whitespace="pre"> &lt;h1&gt;Hello&lt;/h1&gt; </code> <section scrollbar--set width="15rem" height="15rem"> Scrollable content here... </section> </main> </body> </html>
Note: Separate multiple plugins using semicolons (;). Avoid using commas or spaces.

Configuring Plugins

Each plugin can define its own configuration parameters using the {plugin}--conf attribute of the <script> element. You can pass settings such as colors, sizes, or behavior options.

Example:
FTML (index.html) <script src="https://cdn.front.nu/{ma}.{mi}.{pa}/front.js" plugin="syntaxhighlighting;scrollbar" scrollbar--conf="width:auto;radius:3;thumb:orange;hover:black;track:black"> </script>
Note: The {plugin}--conf on the script tag sets the default configuration. Individual elements can override it by passing options directly on the attribute.
Overriding per element:
FTML <!-- Uses default config from script tag --> <section scrollbar--set>...</section> <!-- Overrides specific options for this element --> <section scrollbar--set="thumb:black;track:whitesmoke;hover:gray">...</section>
